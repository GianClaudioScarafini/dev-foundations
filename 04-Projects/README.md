# Projects

These are real projects — not exercises.

The difference: exercises teach you one concept at a time with scaffolding.
Projects force you to make decisions, hit real problems, and ship something that works end-to-end.

Every project here goes on your GitHub. Every project here is something you could demo in an interview.

---

## Rules for every project

1. **TypeScript only** — no plain JS files
2. **Git from day one** — commit as you go, write real commit messages
3. **README in the project** — explain what it is, how to run it, what you built
4. **Deploy it** — Railway/Render for backends, Vercel/Netlify for frontends
5. **No tutorial following** — look things up, but build it yourself

---

## When you're done with a project

- Push it to GitHub as its own repository
- Add the link to your CV
- Be able to explain every decision you made
- Answer: "what would you do differently if you built it again?"
